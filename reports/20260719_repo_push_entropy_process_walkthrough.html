1. 분석이 바뀐 10개의 장면

사람은 질문과 의미를 교정했고, 후속 감사에서는 오른쪽 검열을 처리한 뒤에도 남은 왼쪽 검열 문제를 찾아냈다.

#질문·반박문제분석에 생긴 변화주도
1“멀티 레포 유저가 장기적으로 어떻게 변했지?”출발점월별 actor 활동만으로 프로젝트의 생애를 알 수 없다.Push actor와 repo 구성을 월 단위로 먼저 확인했다.사람
2“작은 테스트 프로젝트가 늘어난 걸까?”가설 확장한 repo actor와 한 actor repo를 곧바로 테스트로 부를 위험이 있었다.목적을 단정하지 않고 활동 분산과 지속성을 보조 지표로 추가했다.사람
3“Push로만 보자.”신뢰 이벤트 제한Watch·Issue 등은 코드 작업과 의미가 다르다.모든 lifecycle 계산을 PushEvent로 한정했다.데이터
4“활성화 패턴을 entropy로 잡자.”지표 발상총 Push만으로 첫날 집중과 장기 분산을 구분할 수 없다.주별 Push 비중의 Shannon entropy를 도입했다.사람
5“90일 전과 이후 8주를 보자.”첫 탐색 설계동일 관찰창은 만들었지만 프로젝트 정의가 임의적이었다.90일 재활성화와 고정 8주 관찰의 4개 유형을 만들었다.스킬
6“협업이냐 아니냐가 핵심이고 종료는 별개 축이야.”결정적 반박처음부터 협업한 뒤 바로 끝난 경우와 혼자 오래 지속한 경우가 비교 불가능한 한 축에 놓였다.4개 상호배타 유형을 폐기하고 협업·종료 원자 컬럼을 분리했다.사람
7“8주가 아니라 28일 무활동이면 이탈이잖아.”운영 정의 복원고정 8주는 실제 lifecycle 계약과 달랐다.연속 Push 간격이 28일을 초과할 때 새 episode가 시작되도록 바꿨다.사람
8“평균과 중앙값 둘 다. threshold는 dynamic.”소비 방식 결정고정 0.25는 품질 기준처럼 굳을 수 있고 평균만 보면 긴 꼬리를 전형으로 오해한다.raw/normalized entropy를 적재하고 평균·정확 중앙값을 함께 제공했다.사람
9“mart로 만들자. 백필 비용도 보자.”분석의 제품화세 분석 SQL에 episode 계산이 중복됐고 정의 변경 때마다 재스캔해야 했다.episode fact와 시작·종료 월별 mart를 만들고 약 $0.18에 전체 refresh했다.스킬
10“1, 2번 문서도 갈아엎자.”전달 정의 교정폐기된 86.7→84.6% 결론이 새 mart 결과처럼 남아 있었다.기존 수치를 폐기된 탐색 결과로 격리하고 중앙값·긴 꼬리 중심으로 다시 썼다.해석 교정

3. 폐기한 결론과 새 결론

폐기한 결론

“초기 종료형이 86.7%에서 84.6%로 감소했다.”

  • 90일 재활성화와 고정 8주라는 탐색 정의에 의존
  • 협업 여부가 종료 분류보다 우선되는 비대칭 case 문
  • 현재 mart의 지표와 직접 연결할 수 없음

새 결론

“새 활동은 늘었지만 첫 주 협업 비중은 조금 낮아졌다.”

  • 중앙 기간 1일, 중앙 entropy 0은 유지
  • 시작 활동 535.2만 → 716.9만
  • 30일 고정창 평균 기간 5.18일 → 5.63일(2025-08 → 2026-05)
  • 종료 월 기준 이후 협업 증가는 왼쪽 검열 재검증 대상으로 보류

5. 왜 episode mart가 필요했나

재사용성

  • threshold 변경 시 fact 재스캔 불필요
  • 협업·종료·entropy를 자유롭게 교차
  • 시작 cohort와 종료 month를 분리

정확성

  • 최근 783.9만 행 provisional 표시
  • 왼쪽 28일 관측이 없는 시작일 제외
  • Push 807,459,224건 합계 보존
운영 함정: 28일 규칙 아래에서 종료된 episode는 모두 churn이다. 따라서 “episode churn rate”는 시간이 지나면 100%로 수렴한다. 종료 건수는 churn month로 운영할 수 있지만 기간·분산·후속 협업의 시계열 비교는 동일한 시작 cohort 관찰창이나 왼쪽 절단 보정이 필요하다.

8. 남은 한계

  • PushEvent만으로 실제 저장소 생성, 프로젝트 목적, 완성도, 사업적 성공을 판별할 수 없다.
  • 현재 episode mart에는 사람뿐 아니라 봇과 자동화가 포함된다. 별도로 만든 제외 차원은 아직 연결 전이다.
  • 종료 월 기간 비교에는 왼쪽 검열이 있어, 고정 관찰창 결과만 핵심 결론에 사용한다.
  • 2026-06 시작 episode의 45.3%, 2026-07의 100%는 기준일 현재 provisional이다.
  • 기간·entropy·협업 전환의 동반 변화는 관찰이며 원인 효과가 아니다.
