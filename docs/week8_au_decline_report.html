Week 8 AU 감소 원인 분석 리포트

데이터 범위: 2025-09-01 ~ 2026-05-21 / 전환점 가설: 2025-12-01

해석 원칙: 이 문서는 AU 감소와 AI/agent 활동량 변화를 같은 기간에 관찰된 현상으로 비교합니다. 상관 또는 동시 관찰은 원인 증명이 아니므로, "AI 활동 증가가 AU 감소를 일으켰다"처럼 단정하지 않습니다.

1. 한 줄 요약

  • 2025-12-01 전후 56일 평균 기준 DAU는 538,776명에서 579,386명으로 7.5% 증가했습니다. 따라서 이 비교만으로는 '12월 이후 내리막'이라고 보기 어렵습니다.
  • 같은 기준에서 WAU는 2,177,615명에서 2,219,759명으로 1.9% 변했습니다.
  • 활성 사용자 1명당 이벤트는 11.6% 변했고, 전체 이벤트 수는 11.7% 변했습니다. 사용자 수와 활동 강도가 같이 움직였는지 확인하는 핵심 단서입니다.
  • AI/agent 후보 repo에서는 최근 성장 지표가 관찰됩니다. 다만 이 리포트는 AI 활동량 증가와 AU 감소를 같은 기간에 관찰된 현상 또는 상관 후보로만 다루며, AI가 AU 감소의 원인이라고 단정하지 않습니다.
  • 월평균 DAU 기준 2025-12는 576,373명이고, 분석 기간 최고 월은 2026-04 705,089명입니다. 12월은 낮아졌지만 2026년 2~4월에 회복/상승 구간이 확인됩니다.

3. 데이터 품질 체크

감소 원인을 찾기 전에 같은 정의와 같은 커버리지의 데이터인지 확인해야 합니다. 특히 2025년 10월 일부 날짜는 이벤트 row와 AU가 비정상적으로 낮아 전체 추세 비교에서 별도 표시가 필요합니다.

  • 일별 mart는 263일 모두 존재합니다.
  • 2025-10-09~2025-10-14에 user-repo-action row가 평소의 20% 미만인 날이 있어, 이 구간은 실제 사용자 감소보다 원천 이벤트 관측/적재 이상 가능성을 먼저 봐야 합니다.
  • 최신일은 2026-05-21입니다. GH Archive 최신 1~2일은 지연 가능성이 있으므로 운영 리포트에서는 보수적으로 해석합니다.

4. 활동 강도 변화

AU가 줄어도 전체 이벤트가 같이 줄지, 아니면 남은 사용자가 더 많이 활동했는지를 분리해서 봐야 합니다. events_per_active_user가 상승했다면 사용자 수 감소와 활동량 집중이 동시에 발생했을 가능성이 있습니다.

5. 이벤트 타입별 변화

7. Trend 후보 검증 테이블

검증 테이블은 trend score가 다음 기간의 관심 repo를 얼마나 잘 맞췄는지 보는 보조 자료입니다. 점수가 좋아도 AU 감소 원인 검증이 아니라, "AI/agent 후보 repo를 고르는 방식이 쓸 만한가"에 대한 평가입니다.

검증 테이블이 없어 모델 품질 비교는 생략했습니다.

8. 외부 맥락

외부 자료는 원인 증명이 아니라 배경입니다. GitHub Octoverse 2025는 GitHub 전체 개발자와 public/open-source 활동이 성장했다고 설명하므로, 이 데이터의 AU 변화가 있다면 GitHub 전체 사용 감소보다 특정 관측 범위, 이벤트 노출, 사용자군 변화로 먼저 보는 편이 안전합니다. 2025년 하반기~2026년 초에는 Codex, Claude Code, GitHub Agent HQ 같은 agentic coding 제품 공개도 이어졌지만, 현재 mart만으로 AI 도구가 AU를 감소시켰다는 인과는 말할 수 없습니다.

  • GitHub Octoverse 2025
  • GH Archive 설명
  • OpenAI Codex 공개, Claude Code on the web, GitHub Agent HQ
  • GitHub Availability Report: October 2025

9. 다음 분석 제안

  • 신규 사용자와 기존 사용자로 나눠 2025-12 이후 감소가 유입 감소인지 재방문 감소인지 분해합니다.
  • AI/agent 후보 repo 활동 사용자와 일반 repo 활동 사용자의 cohort retention을 비교합니다.
  • 자동화 계정, bot 패턴, 극단적으로 높은 이벤트 사용자를 분리해 AU와 이벤트 수를 다시 계산합니다.
  • 데이터 파이프라인 변경, 수집 누락, public event 정책 변화 같은 외부 요인을 같은 날짜 축에 표시합니다.
